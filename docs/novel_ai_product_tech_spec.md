# 小说自动编写工具：产品与技术方案（PRD + TDD）

## 1. 项目定位

### 1.1 愿景
打造一个“**网文题材雷达 + 爆梗热点融合 + 章节级自动创作**”的一体化系统，帮助作者快速从“选题”走到“成稿”。

### 1.2 核心价值
- 自动发现当前高热题材（按阅读量、搜索量、时间趋势排序）。
- 自动提取近期爆梗、热点事件和流行表达，并进行可控融合。
- 基于“题材 + 爆梗 + 用户设定”自动生成章节内容，支持持续连载。

### 1.3 目标用户
- 网文新人作者（缺选题、缺大纲）。
- 工作室编辑（批量评估题材可行性）。
- 内容运营人员（热点借势、快速上新）。

---

## 2. 产品目标与边界

### 2.1 业务目标（建议）
- 选题命中率：进入“中高潜力池”的选题，7日内验证通过率 > 30%。
- 生产效率：从“选题到首章”时间降低到 30 分钟以内。
- 可读性指标：章节平均完读率、停留时长高于历史人工基线。

### 2.2 第一阶段边界（MVP）
**先做**
1. 热门题材榜单（多源聚合 + 排序）。
2. 热点/梗抽取（关键词 + 事件 + 情绪标签）。
3. 章节生成（支持世界观/人物卡/风格约束）。
4. 人工审核与一键改写。

**暂缓**
1. 全自动日更无人值守。
2. 全平台自动发布。
3. 深度商业化模块（签约管理、收益结算）。

## 3. 功能需求（详细）

## 3.1 热门选题发现系统

### 输入数据
- 小说平台榜单：阅读量、收藏、追更、评论、完读率（可得则接）。
- 搜索平台趋势：关键词搜索指数、增长率。
- 社媒讨论度：话题量、互动量、传播速度。

### 排序维度
- `阅读热度`：真实消费行为（阅读、收藏、追更）。
- `搜索热度`：潜在需求（搜索增长）。
- `时间热度`：近期加速度（24h/72h/7d斜率）。

### 排序公式（示例）
`HotScore = 0.45 * ReadScore + 0.35 * SearchScore + 0.20 * TimeScore`

可进一步加入：
- 去水分因子（异常流量惩罚）。
- 竞争强度（同题材供给过多时降权）。
- 新颖度（同质化惩罚）。

### 输出
- TopN 题材榜单（总榜 + 男频 + 女频 + 细分标签榜）。
- 每个题材的“机会解释”：为什么热、热多久、是否在上升。

---

## 3.2 爆梗与热点雷达

### 功能点
1. **热点检测**：识别突发事件、娱乐热点、社会议题。
2. **爆梗抽取**：提取高传播短语、梗模板、语义变体。
3. **生命周期识别**：萌芽期 / 爆发期 / 衰退期。
4. **题材映射**：把热点映射到可写题材（如“职场逆袭”“都市悬疑”）。

### 关键输出结构
- 热点词：关键词 + 强度 + 时间窗。
- 热点事件摘要：一句话背景 + 风险等级。
- 梗可用性建议：可用场景、禁用场景、可替代表达。

### 安全策略
- 自动过滤政治高风险、违法违规、明显侵权内容。
- 对“真实灾难/敏感事件”默认降级处理，避免不当娱乐化。

---

## 3.3 章节自动生成引擎

### 输入
- 选题（如：都市异能 + 逆袭）。
- 热点包（梗、事件、流行语）。
- 创作约束（主角设定、世界观、文风、长度）。
- 连载记忆（前文摘要、伏笔清单、人物关系）。

### 生成流程
1. 题材拆解：冲突类型、爽点节奏、受众偏好。
2. 大纲规划：卷纲 → 章纲 → 本章节拍（hook/冲突/反转/钩子）。
3. 热点融合：将热点按“自然度阈值”插入剧情。
4. 草稿生成：先粗稿，再润色改写。
5. 质量检查：逻辑一致性、违禁词、角色不崩、重复率。

### 输出
- 章节正文。
- “本章亮点”与“下章建议”。
- 角色状态更新、伏笔账本更新。

---

## 3.4 工作台与协作能力
- 选题看板：趋势图、爆发提醒、竞品参考。
- 创作台：大纲/角色卡/章节生成/改写同屏协作。
- 审核台：风险词、事实冲突、版权相似度提示。
- 版本管理：章节版本diff、回滚、A/B稿对比。

## 4. 关键盲点与卡点（重点）

## 4.1 数据层盲点
1. **平台数据可得性不稳定**：反爬、接口变更、数据延迟。
2. **热度口径不统一**：各平台阅读/搜索定义不同，难横向比较。
3. **噪声与刷量**：热点可能被营销操纵，误导选题。

**应对建议**
- 建立多源冗余 + 数据质量评分。
- 指标统一标准化（z-score/分位数归一）。
- 增加异常检测（突增突降、机器人行为识别）。

## 4.2 模型层卡点
1. **“会写”不等于“能连载”**：长程一致性难。
2. **热点融合生硬**：梗硬塞导致阅读割裂。
3. **风格漂移**：写到中后期人物口吻跑偏。

**应对建议**
- 分层生成：先结构，后内容，再润色。
- 引入“记忆库 + 角色状态机 + 设定约束解码”。
- 章节后处理：一致性审校模型 + 规则引擎。

## 4.3 业务层卡点
1. **冷启动难**：无历史作者数据时个性化弱。
2. **命中率验证慢**：题材是否能爆需时间验证。
3. **平台规则变化快**：违禁、导向、推荐机制常变。

**应对建议**
- 首期采用“人机共创”，人保底、机提效。
- 小流量实验体系（选题AB、开篇AB、封面AB）。
- 规则中心可配置，按平台切换策略。

## 4.4 合规与伦理盲点
- 侵犯版权（训练语料、改写过近）。
- 真实热点不当使用带来舆情风险。
- 未标注AI生成引发信任问题。

**应对建议**
- 训练与检索语料合法授权。
- 上线相似度检测、敏感内容审查。
- 提供AI参与度说明与审校日志。

## 5. 技术架构设计

## 5.1 总体架构
1. **数据采集层**：榜单/搜索/社媒数据接入。
2. **数据处理层**：清洗、去重、实体归一、时间对齐。
3. **洞察层**：热度计算、趋势预测、热点挖掘。
4. **生成层**：大纲模型 + 章节模型 + 改写模型。
5. **治理层**：审核、版权检测、日志追踪。
6. **应用层**：Web 控制台 + API。

## 5.2 推荐技术栈（分层）

### 前端
- React / Next.js
- TailwindCSS + 组件库（Ant Design 或 shadcn/ui）
- 图表：ECharts

### 后端
- Python FastAPI（AI编排、推理接口）
- Node.js（可选，负责BFF与实时推送）
- API 网关：Kong / APISIX

### 数据与存储
- OLTP：PostgreSQL（项目、角色卡、章节元数据）
- 检索：Elasticsearch / OpenSearch（热点、关键词、全文）
- 缓存：Redis（实时榜单、会话状态）
- 对象存储：S3兼容（稿件版本、报表）
- 数仓：ClickHouse（趋势分析、行为分析）

### 数据处理与调度
- Airflow（离线任务编排）
- Kafka（实时流：热点事件流）
- dbt（指标建模）

### AI 与 NLP
- LLM 推理：可插拔（OpenAI/本地模型/混合）
- 向量库：Milvus / pgvector
- Embedding 模型：多语向量（中文优化）
- 重排模型：Cross-Encoder（提升相关性）

### 模型治理与评估
- MLflow（实验追踪）
- Prometheus + Grafana（服务监控）
- Langfuse / OpenTelemetry（链路追踪）

### 安全合规
- 内容审核服务（自建规则 + 第三方审核API）
- 机密管理：Vault
- 权限模型：RBAC + 审计日志

## 6. 核心算法与策略

## 6.1 热度排序策略
- 指标归一化：按平台分桶归一。
- 动态权重：不同品类可配置权重（女频更重情绪话题，男频更重冲突强度）。
- 时序预测：可用 Prophet/LSTM/Temporal Fusion Transformer 预测短期热度。

## 6.2 爆梗提取策略
- 候选召回：n-gram + 新词发现 + 词频突增。
- 语义聚类：Sentence-BERT 聚类同义梗。
- 爆发判断：环比增速 + 传播广度 + 持续时长。

## 6.3 热点融合策略
- 融合评分：相关性、自然度、受众匹配度三维打分。
- 控制注入：每章限制热点注入次数，避免“热梗堆砌”。
- 时效衰减：热点超过阈值自动降权。

## 6.4 章节生成策略
- 两阶段生成：
  - Stage A：结构化章纲（冲突、反转、情绪曲线）。
  - Stage B：正文写作（按章纲展开）。
- 记忆管理：
  - 短期记忆：近3~5章事件。
  - 长期记忆：人物设定、世界规则、伏笔账本。
- 质量门禁：
  - 一致性检测（人物年龄/关系/地名）
  - 重复率检测
  - 安全审核

---

## 7. 非功能需求（NFR）
- 可用性：核心服务 SLA >= 99.9%。
- 性能：章节草稿（2000字）生成 P95 < 20s（缓存命中下）。
- 扩展性：支持多模型路由、灰度发布。
- 可观测性：请求级 tracing + 关键指标告警。
- 安全性：数据传输加密、敏感字段脱敏。

---

## 8. 版本规划（建议）

### V1（0-2个月）
- 题材热榜 + 基础章生成功能 + 人工改写。
- 手工导入热点词包。

### V2（2-4个月）
- 自动热点抽取 + 融合推荐。
- 连载记忆与设定约束上线。

### V3（4-6个月）
- 全链路评估闭环（选题命中率→章节表现→迭代优化）。
- 多题材模板与多风格引擎。

## 10. 风险清单与预案
1. **数据源中断**：建立备份源与降级策略。
2. **模型成本过高**：模型分级路由（小模型起草、大模型润色）。
3. **内容违规**：前置审核 + 生成后复核双保险。
4. **用户留存低**：加强“可控性”（可编辑大纲、风格锁定、角色锁定）。

---

## 11. 结论
该产品可行，但必须坚持“**数据驱动选题 + 可控生成 + 合规护栏 + 人机协同**”四原则。真正的竞争壁垒不在单次写作，而在：
- 热点判断准确率；
- 长篇连载稳定性；
- 合规与质量治理能力；
- 可持续的作者工作流体验。
